# UX Direction

## Design system

- UI library / component system
- Design tokens (colors, typography, spacing)
- Component conventions

## Layout and navigation

- Shell structure (header, sidebar, main content)
- Routing and page patterns
- Responsive breakpoints

## Interaction

- Forms: validation style, error placement, submit behavior
- Tables: sorting, filtering, pagination patterns
- Feedback: toasts, loading states, empty states, error states

## Accessibility and i18n

- a11y expectations (keyboard, screen readers, contrast)
- Locale / translation approach (if applicable)
